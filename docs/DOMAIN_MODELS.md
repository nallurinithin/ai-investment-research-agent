# Domain Models

## Overview

This document defines the internal domain models used throughout the AI Investment Research Agent.

The purpose of these models is to establish a consistent contract between all layers of the application, regardless of which external providers (FMP, Tavily, Gemini, etc.) are used.

Every external API response must first be normalized into one of these domain models before being consumed by the Investment Research Agent.

---

# Domain Model Flow

```text
External APIs
        │
        ▼
Normalization Layer
        │
        ▼
Domain Models
        │
        ▼
Investment Research Agent
        │
        ▼
LLM (Gemini)
        │
        ▼
Investment Report
```

---

# Design Principles

- Services retrieve raw data.
- Services normalize external responses.
- Agents only consume normalized models.
- External API schemas must never leak into business logic.
- Domain models remain provider-independent.

# 1. ResolvedCompany

## Purpose

Represents the validated company selected by the user before any investment analysis begins.

This model is produced by the Company Resolver inside the Company Data Service.

---

### Schema

```json
{
  "status": "RESOLVED",
  "companyId": "",
  "companyName": "",
  "ticker": "",
  "exchange": "",
  "exchangeCode": "",
  "country": "",
  "currency": "",
  "sector": "",
  "industry": "",
  "isResolved": true
}
```

---

### Possible Status Values

| Status | Description |
|----------|-------------|
| RESOLVED | Exactly one company was identified. |
| AMBIGUOUS | Multiple companies matched the input. |
| NOT_FOUND | No matching public company exists. |
| INVALID | User input is invalid. |
| UNSUPPORTED | Security type not supported in Version 1. |

---

### Produced By

Company Data Service

---

### Consumed By

Investment Research Agent

---

# Guiding Principles

✅ External APIs are implementation details.

✅ Services own data retrieval.

✅ Services normalize external responses.

✅ The Investment Research Agent never consumes raw API responses.

✅ Every decision made by the LLM must be supported by normalized evidence.

✅ Internal models remain stable even if external providers change.
